## What to Publish After Every New Run

1. `outputs/metrics.json`
2. Updated `docs/RESULTS.md`
3. New checkpoint in HF model repo
4. Updated screenshots or GIFs in `visuals/`
5. Space update (`hf_space/`) if demo behavior changed

## Proof Checklist

- Reproducible script and config used for training.
- Exact epoch count, sample count, and device details logged.
- Checkpoint hash or filename published.
- Public inference/demo page active.
- Before/after comparison table in `docs/RESULTS.md`.
